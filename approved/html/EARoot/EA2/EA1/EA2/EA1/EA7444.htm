-- Definition --<br/>In addition to the basic administrative units in feature class EBM_A (XX_A), also units on upper levels of the national hierarchy will be included in the EuroBoundaryMap dataset. For these additional feature classes the term: ‘Regions’ will be used, as in earlier versions of <br/>EBM (SABE).  <br/><br/>-- Description --<br/>Regions are stored as multipart features, it means that all polygons with the same code build <br/>a single feature. Regions referring to levels in administrative hierarchy contain core name  <br/>attribution, but related tables have to be linked for full maintenance of all attributes.  <br/>For administrative units several regions of a country may not cover the whole extend of the <br/>country. This is because some parts of the country may not be subdivided in all hierarchical <br/>levels. Even the regions from the lowest national level may not be covering the whole <br/>country. Refer to the standard feature class EBM_A (XX_A) in such cases. <br/>
